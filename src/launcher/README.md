# Launcher

[简体中文](README.md) | [English](README.en.md)

进程编排与监管模块，负责读取 TOML 配置、按依赖顺序启动所有子进程、监控进程状态并在异常退出时执行整组重启。

## 功能

- 解析 TOML 配置文件，提取各模块的运行参数
- 按固定顺序启动 13 个子进程，包括 state_machine、前后端采集、推理引擎、数据调度与外设服务
- 捕获子进程异常退出，先发送 SIGTERM 优雅终止所有子进程（3 秒宽限期），超时后发送 SIGKILL
- 支持可配置的重启策略：`restart_attempts` 为 `-1` 时无限重试，为正整数时限制重试次数
- 为每个子进程自动生成带 UTC 时间戳的日志文件，并执行旧日志滚动清理
- 解析配置时对硬编码的 iceoryx2 IPC 服务名进行警告检测，避免用户误配置

## 命令行参数

| 参数 | 说明 |
| --- | --- |
| `--config <path>` | TOML 配置文件路径（默认 `conf/conf.toml`） |
| `--help` | 显示帮助信息 |

## 设计要点

- **IPC 服务命名**：Launcher 在编译期硬编码所有模块间的 iceoryx2 服务名，不允许通过 TOML 修改。配置文件中出现的 IPC 服务名键会被识别并产生警告。此约束确保了系统的零配置通信拓扑一致性。
- **exec + pipe 错误报告**：子进程通过 `pipe2(O_CLOEXEC)` 将 exec 失败时的 errno 传回父进程，避免静默启动失败
- **SIGCHLD 默认处理**：launcher 将 SIGCHLD 恢复为默认行为（SIG_DFL），通过 `waitpid(WNOHANG)` 主动轮询
- **相对路径解析**：配置文件路径优先相对于启动命令所在目录解析，若不存在则相对于运行时根目录
- **日志隔离**：每个模块独立日志文件，命名格式为 `{UTC时间戳}-{模块名}.log`
